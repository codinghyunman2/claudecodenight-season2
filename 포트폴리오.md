**폴더**: `MarketingAgent`

**기술 스택**

### 8. 콘텐츠(쇼츠) 업로드 자동화
**업무 도메인**: 마케팅
**폴더**: `MktContentAgent`

**문제 정의**
Google Drive에 준비된 게임 쇼츠 영상을 YouTube에 수동 업로드. 파일마다 게임명 포함 여부 확인 + 메타데이터 입력 반복.

**솔루션**
Drive 탐색 → 영상 마지막 프레임 텍스트 추출 → 게임명 검증 → YouTube 자동 업로드 파이프라인.

**핵심 로직**
- `{게임명}_Before` 폴더 자동 탐색
- ffmpeg로 마지막 프레임 추출 → Tesseract OCR (신뢰도 70+) → 미달 시 Claude Vision fallback
- 게임명 포함 확인 통과 시만 업로드, 실패 시 폴더 잔류
- ThreadPoolExecutor로 폴더별 병렬 처리

**기술 스택**
- Claude Code, Github, 슬랙 워크플로우
- Python (multi-agent), Google Drive API v3, YouTube Data API v3
- ffmpeg, Tesseract OCR, Claude Vision API

**실제 절감 효과**
- 업로드 작업: 건당 10분 → 0분
- 오업로드(게임명 불일치) 사전 차단

## 공통 인사이트

### 잘 된 자동화의 공통점
1. **판단 기준이 먼저 문장으로 정리됐다** — "ROAS 30% 하락 시 주의"처럼 내 암묵지가 규칙으로 쓰이기 전까지 코드가 아니었음
2. **실패 처리가 명확했다** — 에러 시 어떻게 할지(슬랙 알림, 스킵, 중단)를 설계에 포함
3. **팀이 쓰는 채널(슬랙)로 결과를 냈다** — 별도 대시보드 접속 없이 기존 루틴에 끼워 넣음

### 실패한 자동화의 공통점
1. **파일명 규칙 없이 자동화 먼저** → 소재 오분류, 이후 파싱 규칙 재설계
2. **API 제약 미파악** → Meta API rate limit으로 데이터 0건 리포트 슬랙 발송
3. **내 판단 기준이 너무 많이 들어간 도구** → 맥락 없이 타인이 쓰기 어려움
