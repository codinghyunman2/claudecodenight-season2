Pre-production · 3 Stages

코드보다 기획에
더 많은 시간을 썼습니다.

STEP 01

문제정의

"왜 자동화하는가? 진짜 병목이 어디인가?"

  • "리뷰 답변 자동화"가 아니라
  • "별점별 톤을 매번 재구성하는 비용"이 진짜 문제
STEP 02

PRD.md

우선 4가지만 명확히.

  • 입력 / 출력
  • 판단 기준
  • 실패 시 처리
  • 변경 이력 관리
STEP 03

CLAUDE.md

프로젝트 루트, 컨텍스트 유지.

  • 도메인 용어
  • 판단 기준
  • 폴더 구조
  • 절대 하지 말 것

"내가 매일 클로드에게 똑같은 설명을 반복하고 있다면, 그건 CLAUDE.md에 들어가야 할 내용."

Ch 04 · Three Pillars
14 / 19 · Three Pillars
Three Pillars

비개발자가 챙겨야 할 3축.

PILLAR 01

접근성

접근성이 최우선. SSOT가 될 수 있어야.

  • 꼭 멋드러진 DB일 필요 없음
  • 스프레드시트로 꽤 많이 처리
  • MCP 연동 시트 = 즉시 반영
  • MVP로 먼저 만들자
PILLAR 02

확장성

MVP 이후를 염두에. 갈아엎지 않을 수준으로.

  • 프로젝트 추가 가능성
  • 지표 추가 가능성
  • 스펙 변경 잦은 건 시트 기반
  • 의외로 시트가 유효
PILLAR 03

보안

사내 프로젝트는 대부분 대외비.

  • Daily 체크 → GitHub Actions
  • Monthly → 로컬
  • 팀 공유 → Vercel + Private
  • OAuth는 비번 + 미들웨어 쿠키도 충분
